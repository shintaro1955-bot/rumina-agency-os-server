# Rumina 代理店開拓OS — 本番サーバー（Phase 1 完成・デプロイ手順）

確定要件（小規模 / おまかせホスティング / サーバDB / メール＋パス認証 / AI全機能 / Slack / 社内限定）に沿った
**1サービス構成**（アプリ＋API＋DB を1つのサーバーで配信）です。

## このサーバーが持つ機能（テスト済み）
- 認証：`/api/auth/register`（最初の登録者は自動で「マネージャー」）/ `/api/auth/login` / `/api/me`
- 共有データ（要ログイン）：`/api/companies`・`/api/events`・`/api/goals`・`/api/reportstate`
- 日報台帳（要ログイン）：`/api/nippo`（POST=提出保存＋Slack投稿 / GET=履歴）
- AI：`/api/ai`（Anthropicプロキシ・キー秘匿） / 発掘：`/api/discover`（Google Places New）
- アプリ本体（`public/`）も同梱配信 → 同一オリジンでCORS/混在なし
- DB：SQLite（`DB_FILE`、本番は永続ディスク `/data` にマウント）

## ローカルで動かす
```bash
cd agency-crawler
# .env に ANTHROPIC_API_KEY / GOOGLE_MAPS_API_KEY (任意で SLACK_WEBHOOK, JWT_SECRET)
npm install
npm start            # → http://localhost:8787
```
ブラウザで開く → 1人目を新規登録（=マネージャー）→ 利用開始。

## 本番デプロイ（Render・おまかせ構成）
1. このフォルダ（`agency-crawler/`）を GitHub リポジトリにする
2. Render → **New > Blueprint** → リポジトリを選択（`render.yaml` を自動検出）
3. デプロイ時に環境変数を入力：
   - `ANTHROPIC_API_KEY`（本番用に新規発行推奨）
   - `GOOGLE_MAPS_API_KEY`（Places API (New) 有効化）
   - `SLACK_WEBHOOK`（任意・日報/アラート投稿用）
   - `JWT_SECRET` は自動生成、`DB_FILE=/data/rumina.db` は永続ディスク
4. 発行URL（https）でチーム全員がログインして利用
   - 社内限定にするなら Render の Access 制限 / 後段で IP 制限 / Basic 認証を追加可

> Cloud Run を使う場合：同じ `Dockerfile` でビルド可。SQLiteの永続化が必要なため、
> 永続ボリューム（または Cloud Run + GCS FUSE / もしくは Postgres へ差し替え）を併用してください。
> Postgres へ移行する場合は `db.js` の各関数の中身だけ差し替えれば、サーバ/アプリ本体は無変更です。

## 次の作業（Phase 2・アプリ側の接続）
現状 `public/` のアプリは**まだ端末ローカル保存／ローカルアカウント**で動いています。
本番サーバーの「ログイン」「共有データ」を使うように、アプリ側を次のように差し替えます：
- 起動時：`/api/me` で認証確認 → 未ログインなら `/api/auth/login|register` の画面に
- データ：`window.storage`(localStorage) → サーバAPI（companies/events/nippo/goals）へ
- これで複数人・複数端末で同じデータを共有できる本番状態になります。
